# Security Policy

## Reporting a vulnerability

If you discover a security issue in the ProxWatch installer or any ProxWatch component, please **do not file a public GitHub issue**.

Instead, email **security@proxwatch.com** with:

- A description of the issue
- Steps to reproduce
- The affected version
- Your name and contact info (optional, for credit)

We aim to acknowledge reports within 48 hours and ship a fix or mitigation as quickly as the severity warrants.

## Scope

- The install script (`install.sh`, `update.sh`) in this repository
- The hosted version at `https://proxwatch.com/install.sh`
- The ProxWatch web application, worker, and licensing service

## Out of scope

- Third-party Proxmox VE issues — report those to the Proxmox project
- Vulnerabilities requiring physical access to the host
- Reports that don't include reproduction steps
